# SIT (StayInTouch) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a single Go binary `sit` with `manager` and `node` subcommands — a lightweight remote-management system over WSS long-lived connections, with async task dispatch, SQLite persistence, REST API, and an embedded MCP gateway.

**Architecture:** Node dials outbound-only WSS to a public Manager; Manager dispatches typed JSON Instructions and receives Notifications. Layered Go packages: `protocol` (pure data contract) → `transport` (pure WSS channel) → `manager`/`node` (business) → `manager/mcp` (MCP Relay). Async Task model backed by SQLite, where the tasks table doubles as the offline queue.

**Tech Stack:** Go 1.22, cobra (CLI), nhooyr.io/websocket (WS), modernc.org/sqlite (pure-Go SQLite, no cgo for easy cross-compile), oklog/ulid (IDs), gopkg.in/yaml.v3 (config), golang.org/x/crypto/bcrypt (password hashing), mark3labs/mcp-go (MCP server).

## Task Ordering (dependency order)

Tasks build bottom-up so each layer is testable in isolation. Commit after every task.

1. **Scaffold** — `go mod init`, cobra root, `sit version`. Acceptance: `go build ./... && ./sit version` prints version.
2. **protocol** — Envelope/Instruction/Notification/Ack + codec. Tests: ms timestamp round-trip, dual-stack addrs round-trip, >1MB frame rejected, version mismatch, kind dispatch.
3. **transport (backoff)** — exponential backoff + jitter. Tests: sequence 1,2,4,8…cap; jitter within ±20%.
4. **transport (pump + conn)** — read/write pump over a websocket conn against an in-memory server. Tests: send/recv round-trip, read limit rejects >1MB, close semantics.
5. **transport (dial)** — multi-endpoint failover + Happy Eyeballs. Tests: first endpoint unreachable → next tried; v6 dead → v4 connects.
6. **transport (server)** — Manager WSS upgrade + auth hook. Tests: missing/invalid bearer → 401; valid → upgraded.
7. **store** — SQLite schema + Store interface CRUD. Tests: node upsert, credential hash+revoke, enroll token one-time, task state transitions, activity append, admin bcrypt.
8. **manager auth** — admin token issue/verify, enroll token mint, credential issue/verify/revoke. Tests: login wrong pw rejected; enroll token single-use; revoked credential rejected.
9. **manager registry** — online map, last_seen, offline on >90s. Tests: register binds session; heartbeat refresh; timeout → offline.
10. **manager dispatcher** — create task, online send / offline queue, ack, result pairing, dedup, deadline expiry, timeout. Tests: offline→queued; online→sent; result→succeeded/failed; dup result dropped; expired; timeout.
11. **manager reports** — route register/heartbeat/result/event into registry+store+dispatcher. Tests: register updates store+activity; result reaches dispatcher.
12. **REST API** — router + middleware + all handlers per 03-rest-api. Tests: login flow; nodes list/detail/rename/delete; POST task returns task_id; GET task polls to terminal; enroll; revoke; mcp:enable/disable.
13. **node identity + bootstrap + config** — node_id ULID persisted, creds 0600, endpoints from yaml. Tests: node_id stable across restarts; creds file perms 0600.
14. **node executor** — shell + predefined, timeout kill (process group), 64KB truncate, deadline check, dedup. Tests: capture stdout/stderr/exit; truncate+flag; timeout kills; deadline→expired; dup→replay.
15. **node reporter + client** — register/heartbeat/result/event; recv→ack→exec→result loop; reconnect re-register. Tests: register addrs enumerated; instruction→ack→result; reconnect re-registers.
16. **MCP gateway** — embedded Streamable HTTP server, node-select from Header/Param, mcp_enabled gate, run_command (sync over Task), get_status, MCP bearer auth. Tests: disabled node→403; run_command returns stdout; get_status returns metrics; bad MCP token→401.
17. **wire manager + node subcommands** — `sit manager --config`, `sit node --config`, `sit node enroll`. Acceptance: e2e — start manager, enroll node, node connects, POST task, poll result succeeds.
18. **deployment artifacts** — systemd unit, launchd plist, sample node.yaml/manager.yaml, cross-compile matrix script. Acceptance: `GOOS/GOARCH` matrix builds green.

Each task = TDD micro-steps: write failing test → run (FAIL) → minimal impl → run (PASS) → `go vet ./...` → commit.
